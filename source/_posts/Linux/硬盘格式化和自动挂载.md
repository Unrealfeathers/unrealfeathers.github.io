## 1. 硬盘格式化

### 1.1 识别新硬盘

首先需要确认系统是否识别到了物理硬盘。输入命令后，在输出列表中寻找没有挂载点且容量符合的新设备。

```Bash
lsblk
```

### 1.2 创建分区

如果是一块全新的硬盘，需要先创建分区表。

```Bash
sudo fdisk /dev/sdb
```

1. 输入 `g` 创建新的 GPT 分区表；

2. 输入 `n` 添加新分区，一路回车使用默认设置即可；

3. 输入 `w` 保存并退出；

### 1.3 格式化

```Bash
# EXT4 文件系统
sudo mkfs.ext4 /dev/sdb1

# XFS 文件系统
sudo mkfs.xfs /dev/sdb1

# Btrfs 文件系统
sudo mkfs.btrfs /dev/sdb1
```
{% note warning %}
格式化 `XFS` 需要安装 `xfsprogs` 工具包，通常系统已预装。未安装则需要手动安装：

```Bash
# 查找软件包
dpkg -l | grep xfsprogs

# 安装xfsprogs
sudo apt update && sudo apt install xfsprogs -y
```
{% endnote %}

## 2. 挂载硬盘

### 2.1 手动挂载

创建一个目录作为挂载点，尝试手动挂载。

```Bash
# 手动挂载
sudo mkdir -p /mnt/disk
sudo mount /dev/sdb1 /mnt/disk

# 检查
df -h
```

### 2.2 自动挂载

为了保证硬盘挂载重启后依然生效，需要修改 `/etc/fstab` 文件。强烈建议使用 `UUID` 而非设备名，因为设备名（如 `/dev/sdb`）在增加新硬盘或更换插槽后可能会发生变化。

#### 2.2.1 获取 UUID

```Bash
# 获取 UUID
sudo blkid /dev/sdb1
```

复制输出中 `UUID="xxxx-xxxx"` 部分的字符串。

#### 2.2.2 编辑 fstab

```Bash
# 备份并编辑 fstab
sudo cp /etc/fstab /etc/fstab.backup
sudo vim /etc/fstab
```

#### 2.2.3 详细配置

将 `UUID` 替换为之前复制的 `UUID`。

```Text
# EXT4 文件系统
/dev/disk/by-uuid/UUID /mnt/disk ext4 defaults 0 2

# XFS 文件系统
/dev/disk/by-uuid/UUID /mnt/disk xfs defaults 0 0

# Btrfs 文件系统
/dev/disk/by-uuid/UUID /mnt/disk btrfs defaults 0 0
```

{% note info %}
可以将 `defaults` 替换为 `defaults,nofail`，这样未查找到设备也不会报错；
{% endnote %}

#### 2.2.4 验证配置

在重启前，执行以下命令模拟挂载。

```Bash
sudo umount /mnt/disk
sudo mount -a
```

{% note danger %}
如果这一步报错，说明 `fstab` 写错了，直接重启可能导致无法开机！
{% endnote %}
